Definition
A tree is a type of data structure in which a parent node links to a set of child nodes. Any node that has children is called an internal node, while nodes without children are called leaf nodes.
Definition
Use in representing data
Trees are often used to model data. The top of the tree contains all the data, while each split will divide the data into smaller groups.

each node represents a test on an attribute, i.e. heads or tails
each branch from that node is the outcome

each leaf node represents a class
Terminology

Root

The first node in the tree. In the above diagram 'Colors' is the root node.

Child & Parent

Any node that has child nodes becomes the parent to those nodes. Leaf nodes are never parents. In the above diagram, 'Blue' is a parent while 'Dark Blue' and 'Light Blue' are its children.
The number of children a node has is called its 'degree'.

Descendant & Ancester

The descendant / ancester relationship is similar to that of child and parent, except that the descendant doesn't need to be an immediate child of the ancester. 'Dark Blue' is a descendant of 'Colors';

Leaf

Any node which doesn't have children. The plural of leaf is 'leafs'.

Internal / Branch node

Any node which does have children.

Height

The height of a tree is the length of the longest path between the root node and a leaf node. In the example above, this would be 2.
